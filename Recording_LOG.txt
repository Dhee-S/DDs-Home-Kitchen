================================================================================
                    DD's HOME KITCHEN - PROJECT RECORDING LOG

PROJECT TITLE: DD's Home Kitchen
PROJECT TYPE: Food Ordering Web Application (SPA)
DEPLOYED URL: https://dhee-s.github.io/DDs-Home-Kitchen/

================================================================================
                              PROJECT ORIGIN
================================================================================

ORIGINAL PROMPT:
Build a food ordering website for a home kitchen business featuring Master 
Cook Ponnukodi S, with menu display, ordering system, and admin management.

INITIAL BUILD:
- The core idea was initially generated using Lovable AI platform
- Lovable provided the basic UI/UX framework with fundamental functions
- The project was then moved to GitHub and downloaded locally for 
  customization and enhancement

AI ASSISTED DEVELOPMENT:
- Primary AI execution: OpenCode AI with MiniMax M2.5 model
- Planning assistance: Google Gemini 3.1 Pro
- Additional execution: Claude Sonnet 4.6 for complex implementations
- All AI models used in combination to plan, execute, debug, and refine

DATABASE & AUTH:
- Supabase used for backend database and authentication
- Custom database schema designed for food ordering system
- User roles: MANAGER, CUSTOMER

BUILD TOOLS:
- npm for package management
- Vite for development and production builds
- TypeScript for type safety
- Tailwind CSS for styling
- Framer Motion for animations

================================================================================
                            TECH STACK
================================================================================

FRONTEND:
- React 18+ (TypeScript)
- Vite (Build tool)
- Tailwind CSS (Styling)

================================================================================
                          KEY FEATURES IMPLEMENTED
================================================================================

CUSTOMER FEATURES:
- Browse menu by categories (Today's Special, Weekly, Full Menu)
- Filter dishes (Veg/Non-Veg, In Stock, Scheduled)
- Add to cart with quantity management
- Place orders with notes
- Payment status tracking
- Order history with status
- User profile management
- Share site with friends
- Dark/Light theme toggle

MANAGER FEATURES:
- Add/Edit/Delete dishes
- Upload dish images
- Hide/Show dishes in menu
- Schedule Today's Special and Weekly items
- Category management
- Order management (Pending, Confirmed, Completed)
- Payment verification (Pending/Paid/Failed)
- Special request handling (Approve/Reject)
- Custom delete confirmations

SYSTEM FEATURES:
- Real-time quantity tracking for scheduled items
- Out of stock display
- Scroll to top on page navigation
- Responsive design (Mobile, Tablet, Desktop)
- HashRouter for GitHub Pages compatibility
- Master Cook Ponnukodi S branding throughout

================================================================================
                        DATABASE SCHEMA (SUPABASE)
================================================================================

TABLES:
- profiles (id, name, phone, address, dietary_preference, preferred_pickup_time)
- dishes (id, name, description, selling_price, categories, dish_type, stock_quantity, image_url, is_available)
- orders (id, user_id, order_code, total_amount, notes, status, payment_status, created_at)
- order_items (id, order_id, dish_id, quantity, unit_price, scheduled_menu_id)
- scheduled_menu (id, dish_id, schedule_date, quantity_available, quantity_remaining, preorder_enabled)
- special_requests (id, user_id, dish_name, quantity, request_time, occasion, notes, status, request_date)
- reviews (id, user_id, dish_id, rating, comment, created_at)
- notifications (id, user_id, title, message, type, is_read, reference_id, created_at)

FUNCTIONS:
- place_order (RPC function for atomic order creation)
- generate_order_code (Generates unique order codes)
